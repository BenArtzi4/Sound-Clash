# Changelog

All notable user-visible changes to Sound Clash. The format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

This project does not currently cut versioned releases; every change lands directly in `main` and ships to `https://soundclash.org` via the deploy hooks. The `[Unreleased]` block below is therefore the canonical "what's currently live" list. Internal refactors, test-only changes, CI tweaks, and doc syncs are deliberately omitted; see `git log` for that level of detail.

## [Unreleased]

### Added

- 2026-07-12: **Admins can scan the catalog for dead YouTube videos.** A new admin endpoint (`POST /admin/songs/check-availability`) checks a page of songs against YouTube and reports which ones have been removed (so they can be fixed or deleted before they turn into a "video unavailable" mid-party). It's deliberately cautious — only videos YouTube confirms are gone are flagged as dead; anything it can't confirm (embed-disabled, region-blocked, or a temporary hiccup) is listed separately as "unknown" for a human to check, never auto-removed.
- 2026-07-11: **One-tap presets when hosting a game.** The "Host a game" screen now has a **Quick start** row — *Everything*, *Israeli Mix*, *80s & 90s Party*, *Movie Night* — that fills in the matching genres (and decades) in a single tap instead of hand-picking chips every time. It's a shortcut, not a lock-in: you can still fine-tune the selection afterwards, and the active preset stays highlighted only while it matches what's picked.
- 2026-07-09: **The final scoreboard now stays on screen when a game ends or times out.** When the host ends a game — or a marathon session hits the 4-hour limit and gets cleaned up — every screen used to snap to a bare "This game no longer exists" line the instant the game data was deleted. Now the host, players, and big display all keep the finishing podium and standings on screen (the host also keeps the "export the songs that played" panel), so nobody loses the result the moment the game wraps up.
- 2026-07-09: **Hosts can no longer be locked out of their own game.** The manager console has a collapsed "Backup host link" disclosure (QR + copyable link). Opening that link on any device — say the hosting phone dies mid-party or the browser data gets cleared — turns that device into the host console for the game. The link carries the host credential, so the console warns to share it only with a co-host. Previously, losing the hosting browser meant the game could never be managed again.
- 2026-07-09: **Games can now be extended past the 4-hour limit.** The host console shows a subtle "Ends at HH:MM" hint that becomes a warning banner in the last 20 minutes with a **Keep playing +1h** button (it also appears if the game has overrun its window but hasn't been cleaned up yet). Each press adds an hour and every screen picks up the new deadline instantly. Previously a marathon game vanished mid-round at the 4-hour mark with no warning.
- 2026-07-05: **The big-screen scoreboard now highlights the podium.** The top three teams get gold, silver, and bronze rows — each with a 🥇/🥈/🥉 medal — so the leaders pop at a glance from across the room. Medals only appear once a team has actually scored, so nobody is crowned while everyone is still tied at 0. Display only.
- 2026-06-25: Shared **soundclash.org** links now unfurl into a rich preview card in WhatsApp, iMessage, Slack, Discord, X, and other apps — the Sound Clash logo, name, and "Buzz in. Name the tune. Win the round." tagline on a branded 1200×630 image, instead of a bare blue link.
- 2026-06-25: Sound Clash is now an **installable app (PWA)** — hosts and players can add it to their phone or desktop home screen and launch it **full-screen, without browser chrome**, with a proper Sound Clash icon. Nothing changes for people who keep using it in a browser tab; the join-by-link flow is unchanged. (No offline mode — the game still needs a live connection.)
- 2026-06-24: Added **26 new songs** to the catalog (999 → 1025) — the most-watched YouTube tracks from eight famous artists, human-reviewed: The Beatles, Pink Floyd (rock), Beyoncé, Rihanna, Lady Gaga (pop), and רביד פלוטניק, טונה (Israeli rap/hip-hop) plus שלמה ארצי (Israeli rock-pop).
- 2026-06-23: Hosts can now **export the songs that played** from the end-of-game screen — open them all as a single YouTube playlist link, or download a self-contained HTML file that lists every song in play order with a clickable YouTube link.
- 2026-06-18: Hosts can now **filter songs by decade** when creating a game — tap any of the 60s–20s buttons (multi-select) alongside genres, and only songs from those decades play (genres **and** decades). Leaving decades unpicked keeps every year, exactly as before. For covers, the decade follows the song's original release year, not the cover's.
- 2026-06-18: Admin song catalog now has an optional **Release year** field (song form + CSV import) and shows each song's year as a **Year column** in the catalog list. This is the groundwork for an upcoming "filter by decade" option when hosting a game.
- 2026-06-18: Added **195 new songs** to the catalog (804 → 999), curated and human-reviewed across every genre — heaviest on Israeli music: israeli-rock-pop (+47), mizrahit (+35), israeli-pop (+21), israeli-rap-hip-hop (+7), plus current 2024–2026 Hebrew hits, and refreshes to pop (+27), electronic (+22), rock (+15), hip-hop (+15), soundtracks (+5), israeli-cover (+2).
- 2026-05-24: How-to-play page now opens with a hero illustration showing the three-screen setup at a glance — host's phone with the manager console, TV/laptop running the display + scoreboard + join QR, and team phones with the BUZZ button — so first-time visitors can see how everything connects before reading the steps.

### Changed

- 2026-07-12: **Dead YouTube videos are now skipped automatically.** Songs the availability scan confirms are gone are flagged and never picked for a round, so a "video unavailable" error can't interrupt the party. The flag heals itself: if the video comes back (or the 404 was a hiccup), the next scan restores the song, and an admin swapping in a fresh video for a song makes it playable again immediately. Flagged songs stay in the catalog for the admin to fix — nothing is deleted.
- 2026-07-11: **Team names are cleaned up before they hit the screen.** Invisible and layout-breaking characters (control characters, right-to-left overrides, zero-width spaces) are now stripped from a team's name when they join, so a name can't scramble the projector display or leave junk in the saved game history. Normal names — including Hebrew, other right-to-left scripts, and emoji — are untouched. (PR #229)
- 2026-07-10: rejoining a game with the same team name now resumes your existing team (same score) instead of creating a duplicate or being blocked — so a player who refreshes or loses their tab can get back in with the same name and keep their points. (PR #210)
- 2026-07-05: **The next song's name appears the instant you advance.** On the host console's fast path (where the upcoming song is already prebuffered), tapping **Next round** now updates the round card to the new song's title/artist immediately — the audio already started instantly, but the card used to keep showing the *previous* song's name for the ~150ms until the server responded. Now the name and the sound land together.
- 2026-07-05: **Clearer waiting/loading states.** The team buzzer now says "CONNECTING…" while it's still opening its live connection (and "RECONNECTING…" if that connection drops before the game starts) instead of the misleading "WAITING for the game to start". The host's **Start game** button reads "Loading player…" while the YouTube player is still initializing, so a disabled button reads as progress rather than broken. The admin **Songs** table keeps its rows on screen (dimmed) while a search/filter/page change reloads, instead of flashing back to skeleton placeholders every time.
- 2026-07-05: **Buttons and the scoreboard no longer jump when a team buzzes.** On the host console the "score it" prompt now lives in a fixed slot above the scoring buttons, so those buttons stay put when a buzz lands (previously the banner appeared and pushed the whole row down mid-tap — easy to fat-finger). On the big display, the post-buzz countdown row's space is reserved for the whole round, so the scoreboard no longer slides down when the timer appears.
- 2026-07-05: **Buttons feel snappier and animations no longer jank.** Pressing the BUZZ button (and the host's scoring buttons) now shows the pressed-in state on the same frame as your tap, then eases back on release. The buzzer's pulsing ring, the display's countdown bar, and the background shimmer were re-done to animate on the graphics compositor instead of repainting each frame, so nothing stutters while a phone is doing the work of buzzing — the ambient background drift now runs only on pointer devices (desktops), and the team-name pill dropped an expensive blur.
- 2026-07-05: **The BUZZ button reacts the instant you tap it.** Pressing BUZZ now flips the button to **BUZZED!** immediately, then to **YOU BUZZED** or **SOMEONE ELSE BUZZED** the moment the server answers — a full network round-trip sooner than before, when it waited for the live broadcast to echo back. The live update still reconciles the button, so the state you see is always correct.
- 2026-07-04: **Joining is snappier and the wait is clearer.** The join screen now downloads the in-game screen in the background while you type your team name and wakes the game server ahead of your tap, so going from "Join" to a live BUZZ button is faster. If the server was asleep and is taking a moment, the button now says "Waking the server — up to 30s…" instead of a silent "Joining…". The Host screen gets the same server pre-warm. A brief page load now shows a small pulsing logo instead of a blank flash.
- 2026-07-04: **Less to download and run before the app is interactive.** The error-reporting and latency-telemetry libraries no longer load up front — error reporting is fetched only after the page has painted and the browser goes idle (early crashes are still captured and replayed), and the latency library is no longer downloaded at all. The core framework code is also split into its own long-lived cached file, so a return visit re-downloads only the small app bundle. Net: the main startup JavaScript bundle is ~60% smaller (gzipped), so the app becomes usable faster, most noticeably on phones.

### Fixed

- 2026-07-11: **Rate limits now count each person separately, so a busy party can't throttle itself.** The server's abuse limits (e.g. "10 new games per minute", "30 team-joins per minute") were being counted against everyone combined instead of per person, because behind the hosting proxy every request looked like it came from the same address. On a lively night that meant a run of legitimate hosts or a big group of players joining at once could hit the limit and get turned away for no reason. Limits are now keyed to each real device, so normal play is never blocked and abuse is still contained. (PR #231)
- 2026-07-09: **A brief connection blip no longer blanks the round's song info.** The big display's title/artist reveal (and the host console's song card after a mid-round refresh) look the song up once per round; a single transient network/database failure used to leave them showing "???" or an empty card until the next round. That lookup now quietly retries with backoff for a few seconds, so a blip heals itself. (PR #194)
- 2026-07-08: **A failed "Next round" no longer silences the room.** If advancing the round fails server-side (network blip, database hiccup) after the next song already started playing, the host console now rolls back to the round you were on — the current song reloads and keeps playing, the song card shows the right title again, and tapping Next round retries with the same already-buffered song. Previously both players ended up stopped and the console showed a stale card until the host fumbled it back to life. Host console only. (PR #193)
- 2026-07-08: **The host's +4 Bonus toast no longer claims success before the server confirms.** Tapping a team in the Bonus picker now shows "Sending +4 to …" immediately and the "+4 bonus" confirmation only once the award actually lands — previously a slow or failed call could leave the host believing a bonus was applied that the room never saw. (PR #193)
- 2026-07-08: **Players now see "This game has ended or expired" when a game hits its 4-hour limit, instead of being silently dumped on the homepage.** The expiry sweep deletes team rows a beat before the game row, which the team screen used to misread as "you were kicked" and bounce the player home with no explanation. A real kick from a live game still redirects home as before. (PR #192)
- 2026-07-08: **A network blip while a screen connects can no longer freeze it for the rest of the game.** If the very first game-state fetch failed (flaky venue Wi-Fi, a mid-connect dropout), the buzzer/display/host screens used to keep their live connection but silently ignore every update from then on — scores and buzzes never appeared until a manual refresh. Live updates are now held and replayed once the state loads, and the screen recovers on its own within the minute. All three screens.
- 2026-07-07: **The music no longer stays silent if the host's phone locks or they switch tabs.** When the host returns to the game tab, the song the browser paused now resumes automatically — unless a team has buzzed and is being scored, in which case it correctly stays paused. Previously the host was left with a silently-paused song and had to skip the round to get sound back. Host console only.
- 2026-07-07: **Deploying an update no longer risks blanking a player's screen mid-game.** Previously, if a new version shipped while someone had the app open, their next navigation (e.g. tapping into a game) could fail to load the updated code and show a blank white page until they manually refreshed. The app now detects this and automatically reloads to pick up the new version — and if anything still goes wrong, it shows a friendly "Something went wrong — Reload" screen instead of a blank page. Updates can now go out during live games.
