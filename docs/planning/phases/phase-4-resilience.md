- **T4.11 · Final board survives delete (F-I-FinalBoard)** ✅ (this PR) — `useGameChannel` now exposes a `finalBoard` snapshot of the last-known `GameState` alongside `state`. During play it tracks every committed state; the expiry sweep cascade-deletes `game_teams`/`game_rounds` and those child DELETEs can land before the `active_games` DELETE that nulls `state`, so mid-teardown states shrink. A **shrinking** update is held when it's a teardown: the game is `ended` (every shrink is the post-end sweep → hold the whole board), OR the game is expired-but-unended AND the shrink is not a lone **kick** (one team removed, ≥1 left, rounds intact). A kick is always honored — even in the overdue-but-unswept window (past `expires_at` but still `playing`) — so a removed team can't linger on the final board; removing the **last** team or dropping a round is unmistakably the cascade → held. (Adversarial review caught the first cut using the clock alone as "teardown", which froze the snapshot over an overdue-window kick; fixed + regression tests added.) Residual, accepted for abandoned games only: a >4h-overrun game the host never ended whose sweep delivers team DELETEs before round DELETEs can show a partial board; the ended-game path and rounds-first / last-team sweeps are fully protected. Held in state (not a ref) so consumers see it reactively; the functional updater returns the same reference on a held teardown-shrink, so React skips the otherwise-extra render. All three pages render from it: Display + Team show the `EndScreen` podium under the "ended or expired" banner (Team's banner suppressed for a cleanly-ended game); Manager keeps the podium **and** the song export (the `songs` table is durable, so the export resolves post-delete). No snapshot (a client that opened onto an already-swept code) falls back to the bare banner — no DB read, no `game_history` UI, zero new infra (button-averse). Pure last-known-state chosen over the optional admin-gated `game_history` read. Tests: 7 hook cases (null-before-seen, delete-holds-board, ended-shrink, kick-during-overdue-honored, last-team-held, rounds-first-held, live-kick-updates) + 3 Team + 3 Display + 3 Manager page cases; `expiration.spec.ts` extended to assert the final board on all three clients. Docs synced: `realtime-design.md` §7, `game-rules.md` §7/§10, `testing-strategy.md`, CHANGELOG.
